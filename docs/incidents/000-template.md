# INC-NNN: Short Title (Past Tense Description)

**Date:** YYYY-MM-DD
**Author:** Name
**Status:** Open | Resolved | Mitigated
**Severity:** P0 (outage) | P1 (degraded) | P2 (minor) | P3 (cosmetic)
**Duration:** X minutes/hours
**Components affected:** ArgoCD / database / observability / etc

## Summary

One paragraph: what broke, who noticed, when it was fixed. Could be read in 30 seconds and convey the whole picture.

## Timeline

All times in <timezone>.

- **HH:MM** — what happened
- **HH:MM** — what was observed
- **HH:MM** — what was tried
- **HH:MM** — what worked
- **HH:MM** — incident closed

## Root cause

What was the actual underlying cause? Resist surface explanations. Ask "why" at least three times.

## Detection

How was this discovered? Self-noticed? Alert fired? User complaint? If self-noticed, what was the engineer doing when they noticed?

## Resolution

Step by step, what was done to fix it. Include exact commands where helpful.

## What went well

- Things that helped the incident resolve faster
- Tooling that worked correctly
- Knowledge that paid off

## What went wrong

- Decisions that led to the incident
- Tools or docs that were missing
- Anti-patterns we used

## Action items

Concrete follow-ups. Each with an owner and target date. Status: TODO / In Progress / Done.

- [ ] **Description** — Owner, target date
- [ ] **Description** — Owner, target date

## Lessons learned

The 2-3 sentence "if I had to do this over" version. The takeaway someone reading this 6 months later should walk away with.
